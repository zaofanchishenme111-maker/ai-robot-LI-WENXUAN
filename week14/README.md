# Week 14 - TurtleBot 单人项目总结

## 项目简介

本项目基于 ROS2 与 Docker 环境，完成 TurtleBot 仿真机器人在迷宫环境中的控制与探索实验。通过手机遥控、局域网通信以及 ROS2 节点交互，实现机器人运动控制与环境探索功能。

---

## 项目目标

- 搭建 ROS2 开发环境
- 熟悉 Docker 容器部署流程
- 学习 ROS2 Topic 通信机制
- 实现手机远程控制机器人
- 完成局域网通信测试
- 实现 TurtleBot 迷宫探索

## 项目流程

- 拉取ROS2镜像

- 启动ROS2节点

- 配置网络环境

### 4. 手机遥控

- 连接手机控制端
- 发布速度指令
- 控制机器人前进、后退、转向
- 测试实时响应能力

### 5. 迷宫探索

- 启动仿真环境
- 运行TurtleBot
- 完成迷宫移动实验

## 遇到的问题与解决方案

### 网络连接失败

原因：设备不在同一局域网。

解决：重新配置网络并检查IP地址。

### ROS2节点无法发现

原因：ROS_DOMAIN_ID不一致。

解决：统一ROS_DOMAIN_ID配置。

### 控制延迟

原因：无线网络不稳定。

解决：优化网络环境并重新连接。
